# Markdown 模板与校验规范

## 输出模板

```markdown
# [页面标题]

> 来源： [URL]
> 日期： YYYY年MM月DD日

---

[正文内容]

---

*内容由 AI 提取，不能完全保障原始内容的完全准确性*
```

## 结构要求

- 标题层级必须保留
- 列表必须保留顺序或无序关系
- 代码块必须保留正文和语言标记
- 链接必须保留可点击 URL
- 图片应尽量保留在原正文位置

## 标题映射

| 页面结构 | Markdown |
|----------|----------|
| 一级标题 | `# ` |
| 二级标题 | `## ` |
| 三级标题 | `### ` |
| 四级标题 | `#### ` |
| 五级标题 | `##### ` |
| 六级标题 | `###### ` |

## 图片规则

1. 图片优先落盘到文档同级 `attachments/`
2. Markdown 使用相对路径引用，例如：

```markdown
![示意图](attachments/image-001.png)
```

3. 图片必须和正文位置对应
4. 若图片未能成功本地化，不允许插错图；应在原位置插入占位说明
5. 若图片只暴露为 `blob:` URL，应标注其“运行态可见，但未成功本地化”
6. 建议占位格式：

```markdown
[图片占位：该图片在运行态页面中可见，但未成功本地化]
```

## 完整性检查

### 文本完整性

- 不能只有前半段正文
- 不能在目录仍有后续章节时提前结束
- 不能只保留导航、目录、页脚

### 章节完整性

若页面有目录：

- 目录中的核心章节应大致都出现在 Markdown 中
- 若缺失，应在结果中说明“缺失哪些章节”

### 图片完整性

若运行态页面已看到图片：

- 要么成功落盘并引用
- 要么明确说明该图片未成功本地化

### 错配禁止

以下情况视为错误：

- FAQ 章节引用了无关截图
- 上一章节图片被复用到下一章节
- 仅按文件名猜测图片归属

## 建议输出习惯

- 页面是动态文档页时，按章节整理，不要整页糊成大段文本
- 页面是对话页时，保留发言顺序和角色归属
- 页面是知识库/文档站时，优先保证章节完整，其次再补图
